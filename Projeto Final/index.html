Contexto e Cenários de Aplicação

Objetivo do Sistema

O objetivo principal deste sistema é implementar um mecanismo de autenticação biométrica para controle de acesso físico e lógico, utilizando reconhecimento facial, em áreas restritas e na intranet de uma empresa. O sistema visa:

  • Aumentar a segurança: Substituir métodos tradicionais de autenticação, como senhas e cartões de acesso, por um método mais confiável e seguro, reduzindo o risco de acessos não autorizados.
  • Melhorar a praticidade: Proporcionar uma experiência de acesso mais rápida e conveniente para os funcionários autorizados, eliminando a necessidade de lembrar senhas ou portar cartões.
  • Agilizar o gerenciamento de acessos: Facilitar a administração de acessos, permitindo que os administradores concedam ou revoguem permissões de forma rápida e centralizada.

Ações do Sistema

O sistema funcionará da seguinte maneira:

  1. Captura de imagem facial: Uma câmera capturará a imagem facial do usuário em sua aproximação da área restrita ou ao tentar acessar a intranet.
  2. Reconhecimento facial: A imagem capturada será comparada com um banco de dados de imagens faciais dos funcionários autorizados.
  3. Autenticação: Se a correspondência for bem-sucedida, o usuário será autenticado e liberado para entrar na área restrita ou acessar a intranet.
  4. Registro de acesso: O sistema registrará cada tentativa de acesso, incluindo data, hora, nome do usuário e resultado da autenticação.

Interatividade do Usuário

O sistema proporcionará uma interatividade simples e intuitiva para os usuários:

  • Áreas restritas: Ao se aproximar da área restrita, o usuário se posicionará em frente à câmera e aguardará a autenticação. Uma luz verde ou vermelha indicará se o acesso foi liberado ou negado.
  • Intranet: Ao tentar acessar a intranet, o usuário digitará seu nome de usuário e, em seguida, será solicitado a olhar para a câmera para autenticação facial.

Considerações Adicionais

  • Iluminação: O sistema levará em consideração as condições de iluminação para garantir a precisão do reconhecimento facial.
  • Segurança de dados: As imagens faciais dos funcionários serão armazenadas de forma segura e criptografada, seguindo as melhores práticas de segurança da informação.
  • Privacidade: A coleta e o armazenamento de imagens faciais serão realizados em estrita conformidade com as leis e regulamentações de privacidade de dados.
  • Treinamento do sistema: O sistema será treinado com um conjunto de imagens faciais dos funcionários autorizados para garantir a alta precisão do reconhecimento.
  • Monitoramento e manutenção: O sistema será monitorado periodicamente para garantir seu bom funcionamento e atualizado com as últimas tecnologias de reconhecimento facial.

Modelagem funcional do sistema - Diagrama de Fluxo

Explicação do Diagrama de Fluxo

Fluxo de Dados:

1. Funcionário se aproxima da área restrita ou tenta acessar a intranet.
Fluxo: Solicitação de Acesso → Captura de Imagem Facial

2.Captura de Imagem Facial:
Captura a imagem facial do Funcionário.
Fluxo: Imagem Facial Capturada → Reconhecimento Facial

3. Reconhecimento facial ompara a imagem capturada com o Banco de Dados de Imagens Faciais.
Se houver correspondência:
Fluxo: Correspondência Encontrada → Autenticação
Se não houver correspondência:
Fluxo: Reconhecimento facial

4.Autenticação:
Se autenticado, libera o acesso ao Funcionário.
Fluxo: Acesso Liberado → Registro de Acesso

5.Registro de Acesso:
Registra a tentativa de acesso, incluindo data, hora, nome do usuário e resultado.
Fluxo: Dados do Acesso → Armazenamento no Registro de Acesso
